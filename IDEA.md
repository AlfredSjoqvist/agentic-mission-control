# IDEA.md — FireSight: Spatial Agentic Mission Control for Wildfire Response

## Target Track
**Best Agentic Mission Control with PICO**

## One-Liner
An immersive VR command center on PICO where an incident commander stands inside a world-model-generated 3D terrain, orchestrating AI agents (drone swarms, evacuation routing, fire spread prediction, resource deployment) through voice and gesture — replacing today's fragmented radio-and-flatscreen wildfire response with spatial situational awareness.

---

## The Problem (Real, Urgent, Personal to SF Judges)

Today's wildfire Incident Command System (ICS) was designed in the 1970s. Incident commanders coordinate drone swarms, ground crews, helicopter drops, and civilian evacuations using **radios, flat maps, and whiteboards** while staring at 2D screens. The US Forest Service flew 17,000+ drone missions in 2024 alone — but the command layer hasn't caught up.

Key pain points:
- **No spatial awareness**: commanders can't intuitively see terrain, fire spread, and agent positions in 3D
- **Fragmented data**: drone feeds, weather, terrain, and crew positions live in separate systems
- **Airspace chaos**: 1.19M unauthorized drone violations detected in 2024; mixed crewed/uncrewed coordination is still manual and radio-based
- **Information overload**: a single Type 1 incident can involve 1,000+ personnel across dozens of agencies

The missing piece isn't more drones or better AI — it's a **spatial command layer** that lets a human commander see everything, understand everything, and direct everything from one immersive interface.

---

## The Solution: FireSight

Put on a PICO headset. You're standing inside a photorealistic 3D world model of the fire zone, generated by the **World Labs Marble API** from satellite/aerial imagery. Around you float spatial panels — each one a live AI agent you command.

### Core Agents (built with OpenClaw multi-agent orchestration)

| Agent | Role | Visual in VR |
|-------|------|-------------|
| **🔥 Pyro** | Fire spread prediction — ingests terrain, wind, vegetation data to project fire movement on the 3D world model | Animated fire spread overlay on terrain, color-coded by time (red = now, orange = 30 min, yellow = 1 hr) |
| **🛸 Swarm** | Drone fleet coordinator — manages reconnaissance drone positions, camera feeds, and coverage gaps | Miniature drone icons on the world model + a floating panel showing live feeds |
| **🚨 Evac** | Evacuation routing — calculates optimal civilian exit routes based on fire spread, road status, population density | Glowing route lines on the world model, red for blocked, green for clear |
| **📦 Deploy** | Resource deployment — tracks and suggests positioning for fire crews, water tankers, helicopters | Unit icons on terrain + a resource status board panel |

### Interaction Model
- **Voice**: "Pyro, project fire spread at 25 mph wind from northwest" → fire overlay updates on terrain
- **Gesture/Gaze**: Point at a ridge on the world model → context menu: "Send scout drone" / "Check evacuation route" / "Deploy crew"
- **Spatial panels**: Grab, reposition, resize agent panels around you. Pin critical feeds to your peripheral vision.
- **Timeline scrubber**: Slide through predicted fire states (now → +1hr → +3hr → +6hr) to see how the situation evolves

### Tech Stack
- **Frontend**: WebSpatial (React + CSS 3D transforms) → runs natively on PICO as spatial app
- **World Model**: World Labs Marble API → generates 3D terrain as Gaussian splats, exportable to viewer
- **Agent Layer**: OpenClaw multi-agent routing → orchestrates specialized agents with shared world state
- **Voice Input**: Web Speech API or Whisper → parsed into agent commands
- **Backend**: Node.js or Python FastAPI → WebSocket connection to headset
- **Fire Simulation**: Lightweight cellular automata on terrain mesh (wind + slope + fuel type)

### Build Plan (~35 hours)

| Phase | Hours | Deliverable |
|-------|-------|-------------|
| **1. World Model Foundation** | 0–6 | Marble API generating terrain from wildfire zone imagery. Viewable as Gaussian splat in WebXR on PICO. |
| **2. Agent Skeleton** | 6–12 | OpenClaw orchestration with 4 agent stubs. Each responds to commands and returns structured data. |
| **3. Spatial UI Shell** | 12–18 | WebSpatial app with floating panels + center 3D viewport. Panels show agent status. Runs on PICO. |
| **4. Fire Simulation** | 18–22 | Cellular automata fire spread on terrain mesh. Pyro agent drives it. Visual overlay on world model. |
| **5. Voice + Gesture** | 22–26 | Voice commands routed to agents. Gaze-based terrain selection for contextual actions. |
| **6. Integration + Polish** | 26–32 | All agents feeding live data to spatial panels and world model overlays. Smooth transitions. |
| **7. Demo Rehearsal** | 32–35 | Script the 3-minute demo. Pre-generate fallback world models. Practice transitions. |

### Risk Mitigation
- **Marble API slow/down**: Pre-generate 2-3 terrain world models before demo. Cache them locally.
- **PICO performance**: Keep Gaussian splat resolution manageable. Test on device by hour 12.
- **Agent complexity**: Agents can return mock data with realistic structure if ML inference is too slow. The orchestration layer and spatial UI are the real demo.

## Demo Script (3 minutes)

**0:00–0:30** — "This is how wildfire commanders work today: radios, flat maps, chaos. What if they could stand inside the fire zone?" → Put on PICO headset. World model terrain loads. You're standing above a California hillside.

**0:30–1:15** — "Meet my agents." → Spatial panels appear around you. "Pyro, show me fire spread at current wind." → Fire overlay animates across terrain. "Swarm, where are my drones?" → Drone icons appear on world model, feeds show in panel.

**1:15–2:00** — "Let me direct the response." → Point at a neighborhood. "Evac, route civilians out of Sunset Ridge." → Green evacuation paths light up on terrain. "Deploy, send Engine 7 to block the northern front." → Unit icon moves on world model.

**2:00–2:30** — "Now let me see the future." → Scrub timeline forward 1 hour. Fire spread prediction shows the neighborhood surrounded. "That's why we need to evacuate NOW, not in 30 minutes."

**2:30–3:00** — "Today, incident commanders make these decisions from a folding table with a paper map. FireSight gives them what fighter pilots have had for decades: total spatial awareness. Except the battlefield is a wildfire, and the lives at stake are civilians."
